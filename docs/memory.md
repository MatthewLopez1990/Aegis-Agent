# Memory

Create a confirmed project memory:

```bash
PYTHONPATH=src python3 -m aegis.cli.main memory create project_memory "This repo uses a local SQLite store." --confidence 0.9 --confirmed
```

Search memory:

```bash
PYTHONPATH=src python3 -m aegis.cli.main memory search SQLite
```

Search blends scoped lexical matches with a local deterministic semantic index stored in SQLite. It stays dependency-light and local-first: embeddings are generated by Aegis from normalized token features, not sent to an external provider.

Preview trusted session turns as candidate memories without storing them:

```bash
PYTHONPATH=src python3 -m aegis.cli.main memory session-preview SESSION_ID --owner local-user --scope workspace
```

Commit trusted session candidates while preserving source/provenance:

```bash
PYTHONPATH=src python3 -m aegis.cli.main memory session-commit SESSION_ID --owner local-user --scope workspace
```

Session previews only consider trusted user-directive turns, skip untrusted chat/imported context, block secret-like content, and return review-required candidate records. Session commits rerun the same preview, persist only unblocked candidates, preserve the session/message provenance, and leave records unconfirmed unless `--confirmed` is supplied. CLI, API, TUI, and GUI commit flows can pass selected candidate ids so operators do not have to accept every previewed memory at once.

Explain, export, merge, or expire memory:

```bash
PYTHONPATH=src python3 -m aegis.cli.main memory explain MEMORY_ID SQLite
PYTHONPATH=src python3 -m aegis.cli.main memory export SQLite
PYTHONPATH=src python3 -m aegis.cli.main memory review-queue
PYTHONPATH=src python3 -m aegis.cli.main memory review-digest
PYTHONPATH=src python3 -m aegis.cli.main memory review-escalation --max-age-days 7 --route memory-ops
PYTHONPATH=src python3 -m aegis.cli.main memory review-action MEMORY_ID confirm --rationale "Verified by operator."
PYTHONPATH=src python3 -m aegis.cli.main memory review-batch confirm MEMORY_ID MEMORY_ID --rationale "Verified by operator."
PYTHONPATH=src python3 -m aegis.cli.main memory recertify --dry-run
PYTHONPATH=src python3 -m aegis.cli.main memory recertify
PYTHONPATH=src python3 -m aegis.cli.main memory merge PRIMARY_ID DUPLICATE_ID
PYTHONPATH=src python3 -m aegis.cli.main memory resolve-conflict PRIMARY_ID CONFLICTING_ID keep_primary --rationale "Reviewed current project preference."
PYTHONPATH=src python3 -m aegis.cli.main memory expire MEMORY_ID
PYTHONPATH=src python3 -m aegis.cli.main memory cleanup-expired
```

Review queue, digest, escalation, and recertification commands accept `--scope` in the CLI, API, and TUI so workspace, repo, or imported memory reviews can be handled independently.

Delete memory:

```bash
PYTHONPATH=src python3 -m aegis.cli.main memory delete MEMORY_ID
```

The web GUI exposes the same governed memory safety model for local use:

- Search: `GET /memory?q=...`
- Session memory preview: `GET /sessions/:id/memory-preview?owner=local-user&scope=workspace`
- Session memory commit: `POST /sessions/:id/memory-commit`
- Create: `POST /memory`
- Update: `POST /memory/:id/update`
- Merge: `POST /memory/merge`
- Resolve conflicts: `POST /memory/resolve-conflict`
- Review queue: `GET /memory/review-queue`
- Review digest: `GET /memory/review-digest`
- Review escalation: `GET /memory/review-escalation`
- Review action: `POST /memory/review-action`
- Batch review action: `POST /memory/review-batch`
- Scheduled review digest delivery: `POST /schedules/memory-review-digest`
- Scheduled review escalation delivery: `POST /schedules/memory-review-escalation`
- Recertification: `POST /memory/recertify`
- Expire: `POST /memory/:id/expire`
- Cleanup expired memories: `POST /memory/cleanup-expired`
- Export filtered records: `GET /memory/export?q=...`
- Explain retrieval context: `GET /memory/:id/explain?q=...`
- Delete: `POST /memory/:id/delete`

Web-created memories use `source=web-console`, retain provenance, and still reject secret-like content plus unconfirmed sensitive or uncertain memories.

Conflict resolution is explicit and audited. Operators can keep the primary memory, keep the conflicting memory, synthesize both into one confirmed note, or keep both while tagging them as reviewed. Retired memories are marked deleted and excluded from retrieval; kept or synthesized memories receive conflict-review tags and rationale text.

When live model prompts include relevant governed memory, unresolved conflicts in the same query scope are surfaced as a separate uncertainty section with both memory ids and shared terms. Reviewed conflicts are suppressed from that warning, so the model can use the operator's resolution instead of silently blending contradictory notes.

The review queue aggregates unresolved conflict pairs plus low-confidence, unconfirmed, or recertification-due memories. It is available in the CLI, TUI, and web GUI so operators can confirm, update, resolve, or delete questionable records before relying on them in prompts. The review digest summarizes the queue by severity, item kind, and review reason, then lists top sanitized summaries and next actions for quick operator triage. Review escalation filters that same queue to items older than a configurable threshold and renders a sanitized operator message plus next actions for approval-gated routing. Teams can define route policies under `[memory.escalation_routes.<route>]` with `max_age_days`, `limit`, and `scope`, so `memory review-escalation --route memory_ops` can apply the team's threshold without repeating flags. Operators can also create paused scheduled escalation deliveries, approve and activate them, and let `run-due` render the escalation to the selected channel as pending approval. `review-action` directly confirms uncertain records, raising them to at least `0.7` confidence unless a confidence is supplied, clears the `recertification-due` tag when present, or deletes rejected records. `review-batch` applies the same confirm/delete action to multiple memory ids and records both per-memory audit events and a batch summary. Conflict rows still route through explicit audited conflict resolution.

Memory retention is indefinite by default to support durable recall. Admins can configure `[memory] default_ttl_days` or `[memory.ttl_days]` per memory type in `.aegis/config.toml`, and CLI/TUI/API creation accepts an explicit `ttl_days` override for temporary notes. Expired memories are excluded from retrieval and marked deleted by cleanup.

Memory recertification marks confirmed memories whose `last_confirmed_at` is older than the configured age threshold with `recertification-due` instead of deleting or rewriting them. The next review queue lists those records with `stale_confirmation` so an operator can reconfirm or delete them. Manual recertification is available from the CLI, TUI, API, and web GUI; `--max-age-days` or `max_age_days` provides a one-shot override. Use `--dry-run` or `dry_run: true` to preview the exact stale records and cutoff policy without tagging them for review. The local background maintenance worker uses the configured recertification policy when `aegis serve` is running.

Configure recertification policy in `.aegis/config.toml`:

```toml
[memory]
default_recertification_days = 90

[memory.recertification_days]
episodic_memory = 30
procedural_memory = 0
```

When `aegis serve` is running, the local background maintenance worker also marks expired memories deleted. Manual cleanup remains available from the CLI, TUI, and web GUI; background cleanup logs audit evidence only when it actually expires records.

Self-repair proposals retrieve related confirmed procedural memories tagged `self-repair`. Generated repair candidates include those memories in the isolated patch plan as redacted advisory evidence with memory ids, source, confidence, and summary; they are not treated as instructions or approval to mutate the workspace. Synthesized repair prompts also include the same governed memory context while preserving prompt-artifact checksums and review gates.

Memory is evidence, not authority. Use provenance and confidence when deciding whether to rely on it.
